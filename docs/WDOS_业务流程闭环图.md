## 二、按业务场景拆解的流程

### 场景 1：正常接单处理（最常见，占 90%）

```
CRIP 报警 → Callback → 去重 → 无抑制 → 路由 → 生成工单
→ 小程序通知 → 一线接单 → 处理 → 提交 → 完成 → 解锁
时间线: 约 45-60 秒
```

### 场景 2：重复报警抑制

```
CRIP 报警 → Callback → 去重 → 发现同摄像头+同算法有未处理工单
→ 抑制（追加到原工单，count+1，更新截图）
→ 不生成新工单，不推送通知
时间线: 瞬时
```

### 场景 3：锁定后新报警

```
CRIP 报警 → Callback → 去重 → 点位已锁定
→ 仅记录 raw_alarm，suppressed=true
→ 不生成工单，不推送通知
→ 等待原工单超时 → 逐级上报 → 处理完成 → 解锁
→ 解锁后推送抑制期间报警摘要
时间线: 分钟-小时级
```

### 场景 4：超时逐级上报

```
工单生成 T=0
  T+30s  未接单 → 上报班长（强提醒）
  T+120s 未接单 → 上报经理 + 触发锁定
  T+300s 未接单 → 上报总监

  [假设 T=25s 接单]
  T=A+150s 未处理 → 上报班长
  T=A+300s 未处理 → 上报经理
  T=A+600s 未处理 → 上报总监
```

### 场景 5：转交工单

```
一线接单 → 发现超出管辖范围 → 转交给赵六(班长)
→ 工单回到待接单 → 赵六收到通知 → 赵六接单 → 处理 → 完成
```

### 场景 6：排班自动匹配

```
管理员导入排班表 → 系统解析
→ 6月14日白班 停车场 张三(值班)
→ 当天停车场报警 → 优先分配给张三

[张三休假] → 自动跳过 → 分配给同组李四
```

### 场景 7：领导看板下钻

```
总监登录 → 看到四部门概览 → 点击安保部
→ 进入安保部经理视图 → 看到停车场超时 5 条
→ 点击停车场 → 进入赵六领班视图
→ 看到张三 2 条超时 → 点击张三
→ 进入张三个人工单列表
```
